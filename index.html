Defining and Importing a Class

To organize our code better, we can define the Guitar class in a separate JavaScript file and then import it into our main script.

Step 1: Create a Separate JavaScript File

Save the following code in a new file called Guitar.js:

// Guitar.js
export class Guitar {
    /**
     * Creates a new Guitar instance.
     * @param {string} make - The brand of the Guitar (e.g., "Gibson").
     * @param {string} model - The model of the Guitar (e.g., "SG").
     */
    constructor(make, model) {
        this.make = make; // Assigns the brand to the object
        this.model = model; // Assigns the model to the object
    }

    // Method to simulate playing the guitar
    shred() {
        console.log("Shredding...");
    }
}

Step 2: Import the Class in Your Main JavaScript File

Now, in your main JavaScript file (e.g., script.js), import the Guitar class and use it:

// script.js
import { Guitar } from "./Guitar.js"; // Imports the Guitar class from Guitar.js

// Creating instances of the Guitar class
const myGuitar1 = new Guitar("Gibson", "Les Paul");
const myGuitar2 = new Guitar("Fender", "Stratocaster");

console.log(myGuitar1.make); // Output: Gibson
console.log(myGuitar2.model); // Output: Stratocaster

Step 3: Ensure Your HTML Supports Modules

In your HTML file, make sure you include the script with type="module":

<script type="module" src="script.js"></script>

Key Takeaways:

  • Defining classes in external files makes code more modular and maintainable.
  • Use export to make the class available for import.
  • Use import to bring the class into another script.
  • HTML must use type="module" when importing JavaScript files that use import/export.
